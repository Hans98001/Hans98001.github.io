git和svn对比

发表于 2020-10-23 分类于 学习

实习的时候恰好遇到从svn向git迁移，两个正好一起用过就总结一下两种的区别。

SVN和Git最大的区别是SVN是集中式的，必须联网才能工作，而Git是分布式的且支持离线工作。

Git的核心概念是工作流：

  • 工作区(Workspace)是电脑中实际的目录。
  • 暂存区(Index)类似于缓存区域，临时保存你的改动。
  • 仓库区(Repository)，分为本地仓库和远程仓库。

Git提交代码的流程：

  1. git add 从工作区提交到暂存区
  2. git commit 从暂存区提交到本地仓库
  3. git push 从本地仓库提交到远程仓库

下表是svn和git命令对比。

功能 svn git
工作区更新 svn up(date) git pull
添加文件 svn add git add
删除文件 svn rm git rm
查看工作区状态 svn status git status
提交 svn commit -m “msg” git commit -m “msg”/git push
工作区差异比较 svn diff git diff (HEAD)
显示提交日志 svn log git log
